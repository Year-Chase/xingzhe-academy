# 行者学社 UI 标准

> 版本：V2.0
> 日期：2026-06
> 适用范围：微信小程序 + PC 管理后台

## 1. 产品气质

行者学社不是普通活动报名工具，而是城市人主动恢复精神秩序的活动空间。

关键词：

行走 / 山野 / 城市出走 / 马拉松精神 / 徒步 / 秩序感 / 主动恢复 / 高级放松 / 轻户外 / 自我更新 / 同行者

整体感觉：

年轻、克制、松弛、有力量。
像一个高级轻户外社群，而不是团购活动平台、培训课平台、疗愈工具或后台系统。

禁止出现：

泡澡、按摩、躺平、睡觉、香薰、SPA、冥想插画、卡通人物、治愈系小清新、廉价渐变、网红粉紫色、低幼图标、游戏化勋章、排行榜炫耀感。

## 2. 视觉方向

V2.0 视觉不再只追求“克制”，还要增加：

* 年轻感
* 呼吸感
* 轻盈感
* 活动感
* 城市与山野连接感
* 高级主动放松感

页面不能老气，不能像政务系统、后台管理系统、证书系统。

## 3. 色彩体系

### 主色

* Deep Forest：#18231E
* Mountain Green：#3F6B4F
* Moss Green：#78906F
* Trail Sand：#D9C7A3
* Morning Cream：#F7F2E8
* Mist White：#FBFAF6

### 辅助色

* Sky Gray Blue：#DCE6E2
* Clay Orange：#C98255
* Sunlit Gold：#D6A85A
* Stone Gray：#737A72

### 文字色

* Text Main：#18231E
* Text Body：#3E463F
* Text Sub：#7A8178
* Text Weak：#A6AAA2

### 状态色

* Success：#4F7A5A
* Warning：#C98255
* Danger：#B35B4B
* Info：#4F6F7A

### 禁止

不要大面积使用：

高饱和蓝、亮紫、荧光绿、粉色、廉价金色、纯黑大块、刺眼红绿灯色。

## 4. 字体与尺寸

小程序必须使用 rpx 作为主要尺寸单位，避免手机上字体过小。

### 字体

默认：

PingFang SC / system-ui

不要使用书法字体、卡通字体、过度装饰字体。

### 小程序字体层级

* 页面大标题：44-52rpx / 600
* 模块标题：36-40rpx / 600
* 卡片标题：32-36rpx / 600
* 正文：30-32rpx / 400
* 辅助文字：24-28rpx / 400
* 按钮文字：30-32rpx / 500
* 标签文字：22-24rpx / 500

禁止正文低于 28rpx。
重要按钮文字不得低于 30rpx。
证书、行者时光、活动详情页不得使用过小密集文字。

## 5. 自适配规则

小程序端统一使用 rpx。

页面左右安全边距：

32rpx

模块间距：

40-56rpx

卡片内边距：

32-40rpx

按钮高度：

88-96rpx

底部固定区域：

32rpx + safe-area-inset-bottom

禁止：

* 大量 px 固定尺寸
* 字体小于 24rpx
* 卡片贴边
* 按钮被 TabBar 或 Home Indicator 遮挡
* 页面在真机上显得拥挤

## 6. 页面背景

推荐背景：

* Morning Cream：#F7F2E8
* Mist White：#FBFAF6
* 轻微自然渐变

可使用：

```css
linear-gradient(180deg, #F7F2E8 0%, #FBFAF6 45%, #EEF3ED 100%)
```

禁止：

* 纯白到底
* 大面积深色压迫背景
* 廉价彩色渐变
* 花哨图案背景

## 7. 卡片规范

卡片是小程序核心组件。

### 普通卡片

* 背景：#FBFAF6
* 圆角：28-36rpx
* padding：32-40rpx
* 阴影：轻微、柔和
* 边框：可用 1rpx rgba(24, 35, 30, 0.06)

### 活动卡片

必须包含：

* 活动封面
* 活动名称
* 活动类型
* 时间
* 城市/地点
* 强度
* 名额
* 价格或状态

活动卡片必须可点击，点击区域明确。

### 行者之路卡片

必须更有收藏感：

* 更大留白
* 时间感
* 城市感
* 证书入口
* 同行者信息
* 不做成订单卡片

## 8. 按钮规范

### 主按钮

用于：

报名、支付、确认、授权、分享。

样式：

* 背景：#18231E 或 #3F6B4F
* 文字：#FBFAF6
* 高度：88-96rpx
* 圆角：999rpx
* 字号：30-32rpx
* 字重：500

### 次按钮

用于：

查看详情、返回、取消。

样式：

* 背景：rgba(255,255,255,0.65)
* 边框：1rpx solid rgba(24,35,30,0.14)
* 文字：#18231E

### 禁止

按钮不要像后台系统按钮。
不要使用小而窄的按钮。
不要使用强烈红绿蓝按钮。

## 9. 标签规范

标签用于表达身份、活动类型、状态。

### 活动标签

示例：

轻徒步 / 城市行走 / 马拉松精神 / 山野课程 / 精神补给 / 限 20 人

### 身份标签

* 行者
* 分享者足迹
* 行者足迹
* 终身讲者

注意：

活动级讲者不等于用户级讲者。
只有终身讲者或后台配置的长期身份，才可在身份卡片顶部展示讲者类标签。

### 禁止标签

躺平 / 疗愈 / 解压神器 / 放松按摩 / 心灵鸡汤 / 打卡神器

## 10. 图片风格

推荐：

山路、晨雾、树林、城市边界、跑者背影、徒步队伍远景、岩石、泥土、木头、布料、自然光。

图片应低饱和、高质感、有空气感。

禁止：

大笑摆拍、网红露营、瑜伽冥想摆拍、SPA/香薰/按摩、卡通插画、过度滤镜、廉价户外广告感。

## 11. 首页标准

首页目标：

让用户感受到行者学社气质，并快速进入活动。

必须包含：

* 品牌区
* 核心文案
* 行者活动卡片
* 行者之路入口
* 我的入口

首页活动不可因接口为空而空白。
真实接口失败或返回空数组时，必须有 demo fallback 或清晰空状态。

## 首页 Banner 规范

首页允许存在 Banner 区域。

V1：

不开发 Banner 管理后台。

使用静态 Banner 即可。

推荐：

单张 Banner。

展示品牌气质和活动精神。

推荐元素：

山路
树林
晨雾
跑者背影
徒步队伍
城市边界

禁止：

轮播广告
营销弹窗
商城促销
倒计时抢购

Banner 目标：

建立行者学社品牌感。

而不是提升点击率。


## 12. 活动页标准

活动详情页核心不是卖课，而是建立仪式感。

结构：

* 封面图
* 活动标题
* 时间 / 地点 / 强度 / 人数 / 价格
* 活动精神说明
* 路线或流程
* 适合人群
* 注意事项
* 报名须知
* 底部固定报名按钮

文案示例：

这不是一次打卡式徒步。
我们会用一整个上午，把身体从屏幕里带出来。

## 13. 行者之路标准

行者之路不是数据面板。

它回答三个问题：

* 我是谁
* 我经历过什么
* 我去过哪里

结构：

* 我的身份
* 行者时光
* 足迹地图

### 我的身份

顶部卡片应高级、轻盈、有身份感。

展示：

* 昵称
* 加入时间
* 参加活动次数
* 点亮城市数
* 同行者数

如果是终身讲者，展示特殊身份。
普通活动级讲者不在顶部展示讲者标签。

### 行者时光

核心模块。

默认展示最近 10 条。
每条是一次活动回忆，而不是普通列表项。

内容：

* 时间
* 城市
* 活动名
* 身份：听众 / 讲者
* 证书类型
* 同行者数量

点击必须进入活动回忆页。
不允许点击无反应。

### 活动回忆页

不是普通证书页。

必须包含：

* 活动名称
* 时间
* 城市
* 地点
* 活动照片
* 参与人数
* 我的身份
* 同行者
* 证书卡片
* 分享按钮

### 证书

证书要有收藏感，不要像毕业证、奖状、营销海报。

听众证书：

行者足迹

讲者证书：

分享者足迹

两者必须明显不同。
讲者证书更特殊、更有贡献感。

### 地图

地图是辅助模块。

MVP 可使用轻量模拟地图。
必须展示点亮城市，不允许点击无反应。

## 14. 加载态

全页加载：

居中骨架或轻量 loading，不要只转圈。

局部加载：

使用骨架屏。

按钮加载：

按钮禁用，保留宽度。

图片加载：

占位 + fade-in。

## 15. 空状态

列表无数据：

图标 + 主文字 + 副提示 + 下一步操作。

示例：

你还没有行者时光
先去参加一场活动，让城市留下你的足迹。

禁止页面空白。

## 16. 错误态

网络错误：

温和提示，不崩溃。

接口失败：

Toast + fallback 数据或错误卡片。

页面崩溃：

展示错误卡片 + 重试。

禁止：

接口失败后页面白屏。
点击后无反馈。
错误信息直接展示技术栈。

## 17. 登录与授权

小程序不应打开即强制登录。

流程：

打开小程序
→ 用户须知
→ 可浏览一级页面
→ 进入需要身份的下一级功能时
→ 位置授权
→ 手机号快速登录

真机预览时，不可请求 localhost。
手机无法访问电脑 localhost。
开发环境应支持局域网 IP 或 demo/mock fallback。

## 18. 交互原则

* 首页核心入口不可空白
* 卡片必须可点击时有反馈
* 点击后必须进入明确页面或给出提示
* 下一级页面必须有兜底数据或空状态
* 不允许点击无反应
* 不允许页面因接口失败崩溃
* 不允许关键路径依赖 localhost 才能在手机预览

## 19. PC 管理后台

PC 管理后台保持干净、高效、克制。

建议：

Vue 3 + TypeScript + Vite + TDesign Vue Next

后台不追求强视觉，但必须清晰、高效、状态明确。

一级菜单：

* 数据看板
* 活动管理
* 报名订单
* 支付记录
* 退款管理
* 核销管理
* 用户管理
* 内容配置
* 系统设置

## 20. 开发自检清单

每个页面提交前检查：

* 字体是否适合真机，不低于 28rpx 正文
* 首页活动是否有内容
* 点击是否有反馈
* 空状态是否存在
* 加载态是否存在
* 错误态是否存在
* 是否避免 localhost 真机问题
* 是否符合行者气质
* 是否老气、后台感、证书感过重
* 是否存在低级放松元素
* 是否符合产品宪法
