# 行者学社 Admin 管理后台 PRD

## 1. 阶段定位

P2 Admin 是行者学社的 PC 管理后台 MVP。

P2 的目标不是做完整 ERP，也不是做复杂财务后台，而是先让运营者可以完成活动管理闭环。

P2 完成后，系统应具备：

```text
后台创建活动 → 后台发布活动 → 小程序展示活动 → 用户报名支付 → 后台查看报名与签到状态
```

## 2. 产品边界

### P2 要做

* 新建独立 Admin 工程
* Admin 基础布局
* 活动列表
* 活动创建
* 活动编辑
* 活动发布
* 活动下架
* 活动状态查看
* 活动报名人数查看
* 活动报名人员查看
* 订单只读列表
* 用户只读列表
* Admin 与 WeApp 联动验证

### P2 不做

* 复杂 RBAC 权限
* 退款操作
* 发票开具
* 证书生成
* 会员体系
* 邀请统计
* 财务报表
* 数据大屏
* 内容社区
* H5 管理端

## 3. 用户角色

### 运营管理员

负责创建活动、发布活动、查看报名和线下履约状态。

P2 阶段可以先不做真实多角色权限，默认一个管理员身份。

## 4. 页面结构

```text
Admin
├── 登录页
├── 首页 / Dashboard
├── 活动管理
│   ├── 活动列表
│   ├── 创建活动
│   └── 编辑活动
├── 报名管理
│   └── 活动报名人员列表
├── 订单中心
│   └── 订单只读列表
└── 用户管理
    └── 用户只读列表
```

## 5. 页面需求

### 5.1 登录页

P2 可使用 mock 登录。

字段：

* 账号
* 密码
* 登录按钮

P2 不做：

* 找回密码
* 多角色权限
* 验证码

### 5.2 Dashboard

展示基础概览：

* 活动总数
* 进行中活动数
* 已结束活动数
* 报名成功人数
* 今日新增报名数

如果后端暂不支持，可先显示从已有接口统计的数据。

### 5.3 活动列表

字段：

* 活动 ID
* 活动标题
* 城市 / 地点
* 开始时间
* 结束时间
* 名额
* 已报名人数
* 活动状态
* 操作：查看 / 编辑 / 发布 / 下架

状态：

* DRAFT
* PUBLISHED
* CLOSED
* ENDED

P2 可以先映射现有活动状态，不强行重构状态模型。

### 5.4 创建活动

字段：

* 活动标题
* 活动描述
* 活动地点
* 城市
* 开始时间
* 结束时间
* 活动人数
* 活动封面图 URL
* 活动价格
* 活动状态

P2 可以先使用图片 URL，不做真实上传。

### 5.5 编辑活动

允许编辑：

* 标题
* 描述
* 地点
* 城市
* 时间
* 人数
* 封面图
* 价格

如果已有用户报名，P2 暂不做复杂限制，但需要在页面上提示：

```text
该活动已有报名，修改时间、地点、人数会影响用户体验。
```

### 5.6 发布 / 下架

发布后：

* 小程序首页可见
* 小程序详情可进入
* 用户可报名支付

下架后：

* 小程序首页不可见
* 全部活动是否可见由后端状态决定
* 用户不可继续报名

### 5.7 报名管理

入口：

* 活动列表点击“报名人员”

展示字段：

* 用户 ID
* 昵称
* 头像
* 性别
* 报名状态
* 支付状态
* 二维码状态
* 签到状态
* 报名时间
* 签到时间

P2 只读，不做手动改报名状态。

### 5.8 订单中心

P2 只读。

字段：

* 订单 ID
* 活动 ID
* 活动名称
* 用户 ID
* 用户昵称
* 支付金额
* 支付状态
* 支付时间
* 退款状态

P2 不做：

* 发起退款
* 开票
* 对账
* 导出

### 5.9 用户管理

P2 只读。

字段：

* 用户 ID
* 昵称
* 头像
* 性别
* 手机号
* 注册时间
* 最近登录时间
* 参加活动次数
* 累计支付金额

P2 不做：

* 用户封禁
* 用户编辑
* 用户标签
* 用户详情画像

## 6. 验收标准

P2 完成后必须通过：

### Admin 验收

* Admin 可以启动
* Admin 可以登录
* 可以进入活动列表
* 可以创建活动
* 可以编辑活动
* 可以发布活动
* 可以下架活动
* 可以查看报名人员
* 可以查看订单列表
* 可以查看用户列表

### Admin + WeApp 联动验收

* 后台创建并发布活动
* 小程序首页可以看到该活动
* 小程序进入详情正常
* 小程序报名支付成功
* 小程序生成签到码
* 小程序核销成功
* 后台报名列表能看到报名状态
* 后台能看到签到状态

### P1 回归

P2 不得破坏：

* 首页活动列表
* 活动详情
* 报名支付
* 二维码
* 核销
* 已签到状态
* 已参加盖章
* 同行者展示

## 7. 技术原则

* Admin 独立工程：`apps/admin`
* 不使用 `apps/h5`
* 不直接操作数据库
* 通过 backend API 通信
* 不引入复杂状态管理，P2 可使用简单 store / hooks
* 不引入复杂权限框架
* 不破坏 WeApp 已通过闭环
