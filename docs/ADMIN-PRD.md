# ADMIN-PRD

## 行者学社 Admin 管理后台 PRD

### 1. 当前定位

Admin 是行者学社运营管理端，不是用户消费端。

当前版本：V2.7.2 Online Test Release。

线上地址：https://admin.tenselog.cn

Admin 菜单：
- Dashboard 控制台
- 活动管理（+ 创建/编辑/发布/下架/报名记录）
- 订单管理（退款操作）
- 财务概览
- 发票管理
- 用户运营 CRM
- 证书模板管理
- 手机核销

当前已完成的 Admin 能力：
- Admin 真实 token 登录（HMAC-SHA256 自签名）
- JwtAuthGuard 守卫所有 /admin/* 路由
- 无 token 访问返回 401
- fake-token 验证拒绝
- 手机核销页（选择活动 + 输入核销码 + 签到结果）
- POST /admin/activity/:id/checkin（校验 activityId 归属）
- 订单列表展示用户昵称（可点击跳转 CRM）/ 活动名称
- 用户 ID 缩略展示
- CRM 用户详情头像使用 assetUrl()
- 表格横向滚动

---

### 2. 活动管理

已完成：活动列表、创建活动、编辑活动、发布活动、下架活动、活动状态展示、报名人数展示

**V2.5 增强**：
- 创建/编辑活动六组表单：基础信息、时间与名额、价格与支付、报名信息收集、活动群二维码
- 报名信息收集：6项 checkbox（真实姓名/手机号/身份证号/出发城市/交通工具/房间偏好），身份证号敏感提示
- 活动群二维码：群类型下拉、URL 输入、预览
- 支付模式：FULL/PREPAY 切换，PREPAY 时展示预付/后付金额和后付日期
- 活动回忆：从主表单移出，列表独立"回忆"按钮 → 弹窗
- 活动详情报名信息展示：手机号脱敏、身份证号脱敏

活动回忆规则：建议活动结束后维护，未结束活动也可编辑。支持图片上传（复用已有封面图上传接口）和文字。保存后必须通过 /admin/activity/:id 回读确认。

**待后续增强**：
- 活动模板（V2.5 未做，后续评估）
- 活动到期自动结束
- 活动实际收入/参加人数/退款人数聚合展示

**活动列表字段（V2.5B）**：ID、标题、Slogan、省份、城市、报名开始/截止、活动时间、报名/名额、价格、支付模式、信息收集状态、活动群状态、状态、操作

### 3. 报名信息收集配置

Admin 可勾选：真实姓名、手机号、身份证号、出发城市、交通工具偏好、房间偏好。

勾选身份证号时显示提示："身份证号属于敏感信息，仅在保险、住宿、实名核验等确有必要时使用。"

保存后必须可通过 /admin/activity/:id 回读完整配置。取消全部勾选应保存为空数组。

---

### 4. 活动群二维码配置

字段：groupQrType（NONE/WECHAT_GROUP/WECOM_GROUP/WECOM_LIVE_CODE）、groupQrImageUrl、groupQrTitle、groupQrDescription。

默认值：groupQrType=NONE、groupQrTitle=加入活动群、groupQrDescription=活动通知、集合安排和现场事项将在群内同步。

V2.5B 开发阶段使用 mock URL 测试，生产前接真实上传/存储能力。不接企业微信 API、不自动建群/拉人/退群。

---

### 5. 活动详情报名信息展示

活动详情抽屉/报名记录区域展示：

- 真实姓名
- 手机号（脱敏展示，如 138****0000）
- 身份证号（必须脱敏，兼容 15位/18位/17位+X/x）
- 出发城市
- 交通工具偏好
- 房间偏好
- 信息确认时间

点击报名用户姓名应可跳转到 CRM 用户详情页（后续实现）。

没有 ActivityRegistrationInfo 时显示 `-`，不得页面报错。

### 8. 手机核销页（V2.7.1 新增）

路径：/checkin（MobileCheckin.vue）

功能：
- 从已发布活动列表中选择当前活动
- 输入或粘贴核销码
- 调用 POST /admin/activity/:id/checkin
- 签到成功后显示结果
- 展示最近 20 条核销记录
- 手机优先设计（max-width: 480px）

后端校验：
- 核销码不存在 →"无效二维码"
- 核销码不属于当前活动 →"该核销码不属于当前活动"
- 已签到 →"该核销码已签到，请勿重复核销"
- 已失效 →"该核销码已失效"

### 9. CRM 用户详情订单记录

用户详情页展示该用户所有订单（包括 0 元订单），按 userId 全量查询。

订单记录列表展示：ID / 活动ID / 金额 / 状态 / 时间

0 元订单显示 ¥0.00，状态为 PAID，正常展示不隐藏。

### 10. Admin UI 原则

- 干净、高效、克制
- 状态中文、金额 ¥xx.xx、空值 `-`
- 列多时允许横向滚动
- 不显示 undefined/null/NaN
- 状态明确、颜色统一
- 运营工具优先清晰稳定
- 手机核销手机优先
