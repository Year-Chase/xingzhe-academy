行者学社 PRODUCT
1. 产品定位

行者学社不是普通活动报名工具，也不是单纯后台管理系统。

行者学社是：

活动报名支付系统 + 精美证书传播系统 + 克制型用户运营系统。

当前产品形态：微信小程序 + PC Admin 管理后台 + NestJS 后端 + MySQL + 腾讯云线上环境。

当前商业定位：活动组织、报名、核销、证书、会员运营系统。

它通过线下活动、证书、足迹、地图和同行关系，帮助用户记录行动经历，获得精神满足，并愿意分享给别人。

2. 核心飞轮

用户看到活动
→ 报名支付
→ 现场参加
→ 二维码核销
→ 自动生成活动证书
→ 用户分享证书 / 活动
→ 新用户通过分享进入活动
→ 新用户注册并绑定活动级邀请关系
→ 新用户报名支付
→ 后台统计用户、订单、邀请、证书、城市、收入
→ 运营基于数据做克制、温和、长期的用户运营。

7. 报名支付

支付模式：

全款（mock 支付，当前主流程）
预付+后付（字段已预留，未完整实现）

当前 V2.7.2：API 走 https://api.tenselog.cn，报名支付合并为 enroll-pay（POST /activity/:id/enroll-pay），0 元订单生成真实订单记录。当前不接真实微信支付。

8. 二维码与核销

规则：

支付成功生成报名二维码。
现场扫码核销。
Admin 手机核销页（/checkin）支持选择活动 + 输入核销码。
POST /admin/activity/:id/checkin 校验 activityId 归属。
核销后二维码变为已核销（USED）。
已核销不可重复核销。
核销通过 ActivityFlowService.checkinForActivity() 流转。

## V2.8-Final 产品完成范围

- 报名补充资料：活动可配置标准补充字段，用户报名自动带出常用资料，并保存本次报名快照。
- Admin 订单详情：订单、活动、用户、金额、后付款、退款、发票与真实时间线集中查看。
- 安全二维码：普通用户只查看自己的真实签到二维码，页面不展示明文 token。
- 工作人员核销：用户类型为“工作人员”的小程序用户可进入工作人员工具，选择活动和阶段，支持单次扫码与连续扫码。
- QR 阶段：支持 LEGACY、FULL、PREPAY、POSTPAY；后付款完成/免除可生成 POSTPAY 二维码。
- 明确未完成：真实微信支付、真实微信退款、订阅消息、活动级工作人员权限、签到大屏、支付账本。
